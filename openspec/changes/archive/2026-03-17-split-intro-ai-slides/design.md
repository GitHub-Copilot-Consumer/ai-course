## Context

`site/content/lessons/ch-intro-ai.md` 是課程導言章節，目前內容無任何 `---` 分隔符。根據 `chapter-presentation-mode` spec，簡報模式以 `SLIDE_SEPARATOR_TAG = 'HR'`（即 Markdown 的 `---`）為邊界切分投影片。因此，在文章中加入 `---` 是實現多頁投影片的唯一機制。

## Goals / Non-Goals

**Goals:**
- 在 `ch-intro-ai.md` 加入 11 個 `---` 分隔符，將文章切分為 12 頁投影片
- 每頁投影片對應一個獨立的概念單元，不讓單頁內容過多
- 不更動任何文字內容、圖表或 front matter

**Non-Goals:**
- 修改 `chapter-presentation-mode` 的 JS 邏輯或 CSS 樣式
- 調整 Hugo 建置設定或 layout
- 更改文章的文字內容

## Decisions

### 決策：12 頁的切分方式

依文章章節與內容密度，選擇以下 12 頁切分，在每個 `---` 位置插入分隔符：

| 頁碼 | 標題/內容 | 切分理由 |
|------|-----------|---------|
| 1 | 標題頁（front matter title + 導言 blockquote + 章節問題列表） | 開場，內容少，適合單頁 |
| 2 | Token 與 Context Window（含 ASCII 圖） | 核心概念，圖表需要空間 |
| 3 | 機率生成機制 + 溫度參數（含比較表） | 兩者都在講「輸出穩定性」，合併仍精簡 |
| 4 | 什麼是 Agent（定義 + Agent Loop ASCII 圖） | 公式 + 圖表，內容充足 |
| 5 | ReAct 模式 + Memory 類型（含比較表） | 都是 Agent 運作細節，合併合理 |
| 6 | 什麼是 Coding Agent + 三種專屬工具 | 定義 + 工具列表 |
| 7 | 與一般 Chat 的差異 + 為何需要規格 | 對比 + 引出 SDD，邏輯上承接 |
| 8 | 主流工具前言 + 模型支援光譜（ASCII 圖） | 光譜圖需要整頁空間 |
| 9 | 功能比較表 | 表格內容多，獨立一頁 |
| 10 | 觀察重點：多模型支援本質差異 + Subagent 架構差異 | 兩個觀察點合併 |
| 11 | 觀察重點：Fallback + 本課程的選擇 | 結論性內容，自然收尾 |
| 12 | 小結（ASCII 圖） | 最後一頁，總結脈絡 |

## Risks / Trade-offs

- [Risk] 插入 `---` 後，網頁非簡報模式下會多出水平分隔線，影響閱讀版式 → 接受，這是簡報切分機制的必要代價，現有其他章節亦同
- [Risk] 投影片內容密度不均（如功能比較表一頁、標題頁一頁） → 接受，符合授課節奏設計
